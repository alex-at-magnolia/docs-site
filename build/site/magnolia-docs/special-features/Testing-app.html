This page explains how to set up and run a single test in one page in Magnolia using the Testing app.

Creating a new test

When you create a new test, you are guided through the process in the Testing app. Each step has a dedicated tab where you define:

  • Basic setup

  • Goals

  • Variants

  • Variant weighting

  • Audience

  • View results of a running test

Once you have created and stated your test, the app shows you where you are in the duration and how much time is remaining to get significant results. The duration estimate typically includes estimated traffic on the site.

At any point during the creation of the test, you can save it and carry on later. You can always see the state of your test in the Testing app.

Basic set up

To create a test:

  1. In the testing app, click Create new test.

  2. In the Setup tab, enter a Name and Description for your test.

  3. Select the Original page on which the test is based.

  4. Choose a Start date for the test to be run.

Once you have completed the basic set up, go to the next tab to define a goal for the test.

Goals

To define a goal for your test:

  1. Click the Goals tab of your test.

  2. Enter a Goal name. Where can this be seen later?

  3. Select one of the default goal types from the What to track dropdown. What are the defaults?

Once you have defined a goal for your test, you can move on to the next tab to select the variants to be tested.

Variants

To select and define the different variants involved in your test:

  1. Click the Variants tab of your test.
    There are two variants by default:

    1. The original page you selected in the basic set up is Variant A.

    2. Variant B is a copy of variant A.

  2. Click Rename variant and enter a meaningful name and description for your test variant. For example: XXX

  3. Click Edit variant to make changes to the variant your original will be tested against. You can make your changes directly in the Pages app dialog that opens; you do not have to navigate away from your test.

MVP?? Click Add variant to add more variants to your test.

Once you have edited the variant, you can check the weighting between variants and decide what percentage of your audience you want to dedicate to the test variant.

Weighting

You can see the default weighting applied in the Weighting tab. Weighting defines what percentage of chance a visitor to your site has to be served a variant.

By default in Magnolia, all variants are evenly weighted. This means that if you only have one variant, 50% of the weight is allocated to the variant and 50% to the original. Therefore, a visitor to your site has a 50% chance of being served the variant.

Audience

Define what percentage of your audience you want to allocate to the test. Recommendation? Implications?

Bear in mind that variants are evenly weighted by default in Magnolia. If you choose to allocate 50% of your audience to the test, and your test involves the original page and one test variant (evenly weighted 50% each).

Use the Engagement slider bar to select a percentage of your audience to allocate to the test.

In the Segments section you can further narrow down the audience assigned to the test. Use the drop down to choose the segment for the audience you want.

Test states

Creating a test is a guided multi-step process. At any point during the process you can save your test to complete it later. Once you have finished setting up your test and started running it, you can see its status in the Testing app:

  • Not started - Your test has been created but not started. It may still require some set up.

  • Running - Your test is currently running. An estimate of the remaining time is displayed. You can only edit the test’s name and description at this stage, or pause or abort the test.

  • Paused (variant remains locked) - You have paused your test.

  • Aborted (variant is no longer locked) - You have cancelled your test.

  • Completed (manually by the user) - You reached a significant result and completed your test.
