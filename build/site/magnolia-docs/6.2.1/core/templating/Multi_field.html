Multi field

Multi field is a field type that can render multiple field sets, for example three sets of two text fields. Magnolia provides two implementations of this field type:

  • JCR multi field for JCR-based data sources. Apart from being intended to be used with JCR data, this type is more integrated with the UI owing to its read-write nature. The editor can thus adjust the number of the field sets in a form dialog by clicking either the Add button or the trash bin icon. The layout of the sets can be reordered, using the up and down arrows in the dialog.

  • JSON multi field for JSON-based sources.

Both implementations extend MultiFieldDefinition($type alias multiField).

[.admonition-icon .confluence-information-macro-icon]##

These JCR and JSON multi field definitions are part of the Magnolia 6 UI framework. Their fully qualified class names are info.magnolia.ui.field.JcrMultiFieldDefinition and info.magnolia.rest.ui.field.JsonMultiFieldDefinition respectively.

In the Magnolia 5 UI framework, the field functionally parallel to the JCR multi field is Multivalue field. There is, however, no corresponding 5 UI type for the JSON multi field.

Multi field properties

field

required

Node containing a multi field definition, a field $type (for example the compositeField) and the individual field items.

entryResolution

optional

Implementation class that defines the way the child entries of the multi field should be resolved.

Two configured types are provided - jcrMultiField and jsonMultiField - with the following pre-defined resolvers, respectively:

  • info.magnolia.ui.editor.MultiFieldEntryResolution

  • info.magnolia.rest.ui.field.JsonMultiFieldEntryResolver

orderHandler

optional, default is `info.magnolia.ui.editor.MultiFormDefinition$OrderHandlerDefinition$Noop`

Implementation class that arranges the nodes according to their order and renames them so that the node names reflect the indices correctly.

The jcrMultiField uses the info.magnolia.ui.editor.DefaultJcrNodeOrderHandler.

Currently, the property is not supported for the jsonMultiField.

canRemoveItems

optional

Defines whether the items of a multifield can be removed from a dialog. If set to false, the trash bin that is usually clicked to remove the items is not displayed in the dialog.

By default, the property is set to true in the jcrMultiField type.

See also Common field properties.
