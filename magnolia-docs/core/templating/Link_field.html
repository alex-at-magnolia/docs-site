Link field

LinkFieldDefinition renders a text field and a browse button that allows you to select an item from any configured app or data source. It extends ComboBoxFieldDefinition (Magnolia 6.2.1+).

The link field is used to select targets inside Magnolia (for example, an asset to render an image or a page to tease) where it stores a reference to the selected item.

[.admonition-icon .confluence-information-macro-icon]##

This link field definition is part of the Magnolia 6 UI framework. The fully qualified class name is info.magnolia.ui.field.LinkFieldDefinition.

If you work with the Magnolia 5 UI framework, see Link field for Magnolia 5 UI instead.

JsonLinkFieldDefinition is an alternative that wraps the linkField type and shows an error if a JSON datasource is not available.

Example link field definition

link:

` label: Select image`

` $type: linkField`

` chooser:`

` workbenchChooser:`

` appName: dam-chooser`

Link field properties

chooser

##required

Generic chooser for items of any kind. Configured via info.magnolia.ui.chooser.definition.ChooserDefinition.

info.magnolia.ui.chooser.definition.ChooserDefinition is also used with MoveWorkbenchDefinition.

workbenchChooser

required

Chooser based on a workbench where only one item at a time can be selected. Usually corresponds to an app that uses the same workbench. Configured via info.magnolia.ui.chooser.definition.SingleItemWorkbenchChooserDefinition.

appName

required (unless datasource is used)

Name of the target app (for example, dam-chooser or pages-app) that is used to create the content view to choose the item from. The view is automatically resolved from the app name via info.magnolia.ui.chooser.definition.AppAwareWorkbenchChooserDefinition.

datasource

Connects the target app to a data source for custom implementation. If both datasource and appName are defined, the datasource property will prevail.

Example custom implementation

chooser:

` workbenchChooser:`

` datasource: *someDatasource # here an already defined data source is reused`

` columns:`

` - name: jcrName`

` caption: name`

` $type: jcrTitleColumn`

` - name: value`

` caption: value`

` - name: jcrPath`

` caption: path`

` $type: jcrPathColumn`

actions

optional, default is commit and `cancel`

Defines actions in the dialog. Specified actions are rendered as buttons via ActionDefinition.

Only the actions that are defined will be provided. If no action is specified, the dialog will have commit and cancel actions by default.

name

optional, default is `chooser`

Name of the dialog.

buttonSelectNewLabel

optional, default is `Select new …​`

Button label before the target node is selected. The value is i18n-able.

buttonSelectOtherLabel

optional, default is `Select another …​`

Button label after the target node is selected. The value is i18n-able.

editable

optional, default is `true`

Makes the text box displaying the link editable once a target has been selected.

preview

optional

Renders a preview of the selected content. The preview component typically displays an image thumbnail and some metadata.

implementationClass

required

Any class that implements the info.magnolia.ui.preview.ItemPreviewComponent interface. Examples:

  • info.magnolia.contacts.app.ContactPreview

  • info.magnolia.rest.ui.field.linkfield.JsonItemPreviewComponent

See also combobox field, select field, and common field properties.
